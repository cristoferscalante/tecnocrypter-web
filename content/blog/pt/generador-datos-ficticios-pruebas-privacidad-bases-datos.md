# Testes éticos: geração de dados fictícios realistas para proteger a privacidade

Na era do Big Data e da estrita conformidade com a privacidade, é obrigatório proteger as informações que seus aplicativos manipulam durante as fases de desenvolvimento e demonstração comercial. 

A utilização de dados pessoais reais em ambientes não controlados é uma das causas mais frequentes de multas regulatórias e vazamentos de dados corporativos.

### Mascaramento e geração sintética

Para resolver esse desafio, os engenheiros usam duas metodologias:
1. **Mascaramento de dados:** Criptografe ou ofusque dados de produção existentes.
2. **Geração de dados sintéticos:** Crie registros do zero que imitem o comportamento humano real (como endereços postais válidos, números de cartões de teste e números de telefone fictícios estruturados).

O uso de dados sintéticos garante que seus analistas e testadores de software tenham material realista para trabalhar, sem qualquer possibilidade física de comprometer identidades reais.
